## 6. Push / PR 作成フェーズ

### 6.1 push
```bash
git push -u origin chore/cursor-mcp-config
```
- 新ブランチ `chore/cursor-mcp-config` を `origin` に作成。
- remote から push ルール（merge commit 禁止等）の通知が出たが、bypass されて push 成功。
- （参考）Dependabot が default ブランチに 19 件の脆弱性（high 8 / moderate 8 / low 3）を報告。

### 6.2 発生したトラブルと対処
- `gh pr create` が **2回失敗**（`Head sha can't be blank` / `No commits between master and ...`）。
- 原因調査の結果、本リポジトリには**2つのリモート**が設定されていた:

  | remote | URL |
  |--------|-----|
  | `origin` | `wombat2006/techdev-cursor`（branch を push した先） |
  | `upstream` | `wombat2006/techdev` |

- `gh` がデフォルトで **`upstream`（`wombat2006/techdev`）** を対象にしてしまい、
  そこにはブランチが存在しないため SHA を解決できず失敗していた。
- 対処: **`--repo wombat2006/techdev-cursor` を明示**して再実行 → 成功。
